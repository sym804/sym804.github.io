Case Study: Allure HTML → PDF 변환

Slack 보안 정책으로 HTML 파일 열람 불가 → PDF 자동 변환 시스템 구축

문제 상황

  • Allure 리포트는 JavaScript SPA 기반 → 일반 HTML→PDF 변환 도구로 처리 불가
  • allure-pdf, wkhtmltopdf 시도 → 한글 깨짐 발생
  • Slack 보안 정책으로 HTML 파일 직접 열람 차단 → PDF 변환이 필수

해결 과정 (7단계 시행착오)

  • 1차 allure-pdf: 한글 깨짐 → 실패
  • 2차 Puppeteer: 기존 Python 테스트 자산과의 통합성을 고려해 Playwright Python으로 전환
  • 3차 Playwright 선정: 브라우저 렌더링 방식으로 한글 문제 해결
  • 4차 첫 페이지만 변환됨: Overview/Graphs/Timeline/Behaviors 탭 순회 클릭 로직 추가
  • 5차 여백·레이아웃 문제: A3 가로 + scale 0.9 최적화
  • 6차 시나리오 상세 미표시: node__order 개별 PDF 생성 + PyPDF2 병합
  • 7차 테스트 스텝 미펼침: div.step chevron 클릭 자동화로 완성

결과

  • 일일 자동화 테스트 결과 PDF를 Slack 채널에 자동 전송 — 현재 운영 중
  • 2주 주기 정기 배포 리포트 자동 생성 → 수동 캡처/편집 작업 제거
  • HTML 열람 불가 환경에서도 PDF 즉시 공유 가능 → 리포트 리뷰 소요 시간 50% 이상 단축
  • QA 관점: 기존 도구가 안 되면 직접 만든다 — 7번의 시행착오를 거쳐 운영 수준까지 완성
