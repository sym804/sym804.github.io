YM TestCase는 QA 팀과 개발팀을 위한 셀프 호스팅형 테스트케이스 관리 도구입니다. 스프레드시트처럼 빠르게 작성하고, 대시보드로 진행률을 한눈에 파악하세요.

Why

왜 만들었는가

기존 도구들의 한계를 직접 경험하고, 진짜 필요한 것만 담았습니다.

기존의 문제

  • TestRail — 높은 라이선스 비용, SaaS 종속
  • Excel — 협업 불가, 버전 관리 지옥
  • Kiwi TCMS — 설치 복잡, 느린 UI
  • 자체 개발 — 수개월의 개발 기간

YM TestCase의 해결

  • 무료 오픈소스 (AGPL-3.0)
  • 스프레드시트 스타일 즉시 편집
  • 5분 설치, 즉시 사용 가능
  • 셀프 호스팅으로 데이터 주권 확보
Features

핵심 기능

테스트케이스 작성부터 실행, 분석, 리포트까지 — 전체 QA 워크플로우를 하나의 도구에서.

📋

스프레드시트 편집

AG Grid 기반 인라인 편집. 멀티 행 추가, 벌크 삭제, Undo/Redo, 찾기/바꾸기(정규식) 지원.

🗂️

시트 트리 구조

N-depth 계층형 시트로 테스트 스위트를 폴더처럼 관리. 엑셀 스타일 탭 인터페이스.

⚙️

커스텀 필드

6가지 필드 타입(텍스트, 숫자, 셀렉트, 멀티셀렉트, 체크박스, 날짜). 프로젝트별 동적 추가.

▶️

테스트 실행

테스트 런에서 P/F/B/N 키보드 단축키로 빠르게 결과 기록. 타이머, 첨부파일 지원.

📊

대시보드 & 리포트

Chart.js 기반 진행률 시각화. 라운드별 비교, 회귀 감지. PDF/Excel 리포트 생성.

🔍

고급 필터

AND/OR 다중 조건 필터. 6가지 연산자(=, !=, 포함, 미포함, >, <). 필터 뷰 저장/로드.

📥

Import / Export

Excel 멀티시트 임포트. Jira CSV(35+ 헤더 매핑, CP949/UTF-8 BOM 자동 감지). 벌크 Upsert.

🎯

테스트 플랜

마일스톤 기반 테스트 계획. 릴리즈별 실행 추적과 커버리지 시각화.

🔐

인증 & 권한

JWT + httpOnly 쿠키. 시스템 역할(Admin/QA Manager/User)과 프로젝트 역할(Admin/Tester) 이중 RBAC.

📝

Markdown Import

Markdown 테이블을 TC로 직접 가져오기. # 헤딩으로 시트 분리. GitHub/Notion에서 작성한 TC 바로 import.

🔧

필드 커스터마이징

기본 필드 표시 이름 변경, 숨김 설정. 프로젝트별 독립 관리. 팀 용어에 맞춰 자유롭게 구성.

📋

이슈 관리 체계

심각도 5단계(Block~Trivial), 영역 5가지. GitHub Issue Template으로 양식 강제. 엑셀과 1:1 동기화.

🌐

다국어 지원 (i18n)

한/영 완전 지원. 28개 로케일 파일, 650+ 번역 키. AG Grid 로케일 자동 전환.

♿

접근성 (a11y)

ARIA role/tabIndex, 키보드 네비게이션 지원. 다크 모드 대응.

⚡

성능 최적화

SQL 집계 최대 43배 개선. 6,800 TC / 26,800 결과 기준 최적화 완료. 인라인 자동저장 500ms debounce.
